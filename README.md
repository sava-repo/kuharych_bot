# 🍳 Кулинарыч — Бот для сохранения рецептов из Reels

Telegram-бот, который автоматически извлекает рецепты из видео Instagram Reels, TikTok и YouTube Shorts. Транскрибирует речь, распознаёт ингредиенты и шаги приготовления, сохраняет рецепты в персональную коллекцию.

## Что умеет

- **Сохраняет рецепты из видео** — отправьте ссылку на Reels/Shorts, бот скачает видео, распознает речь и сгенерирует структурированный рецепт с помощью ИИ
- **Каталогизирует по категориям** — рецепты автоматически распределяются по трём категориям: Завтрак, Основное блюдо, Десерт
- **Показывает случайный рецепт** — нажмите кнопку категории в меню, чтобы получить случайный рецепт из коллекции
- **Работает с группами** — создавайте общие коллекции рецептов с друзьями или семьёй
- **Управляет дубликатами** — если рецепт уже есть, предложит перезаписать или сохранить как новый
- **Перемещает между категориями** — любой рецепт можно перекатегоризовать

## Пользовательская документация

### Начало работы

1. Найдите бота в Telegram и нажмите **Start**
2. Бот автоматически создаст личную коллекцию «Личные рецепты»
3. Отправьте ссылку на видео с рецептом — бот обработает его и сохранит

### Сохранение рецепта

Просто отправьте боту ссылку на видео:

```
https://www.instagram.com/reel/ABC123/
```

Бот:
1. Скачает видео
2. Распознает речь из аудио
3. Сгенерирует рецепт (название, ингредиенты, шаги приготовления, категория)
4. Сохранит в вашу активную группу

Если бот уже видел этот рецепт (по ссылке), он сразу добавит его в вашу коллекцию без повторной обработки.

### Меню бота

Внизу чата всегда доступны кнопки:

- **🌅 Завтрак** — показать случайный рецепт из категории «Завтрак»
- **🍽 Основное блюдо** — показать случайный рецепт из категории «Основное блюдо»
- **🍰 Десерт** — показать случайный рецепт из категории «Десерт»
- **👥 Мои группы** — управление группами

### Управление рецептами

Под каждым рецептом доступны inline-кнопки:

| Кнопка | Действие |
|--------|----------|
| 🗑 Удалить | Удаляет рецепт из вашей коллекции |
| 📂 Другая категория | Перемещает рецепт в другую категорию |
| 🎲 Другой рецепт | Показывает следующий случайный рецепт |
| ▶️ Открыть Reels | Переход к оригинальному видео |

#### Если рецепт уже существует

При отправке ссылки на уже существующий рецепт бот предложит:

- **✏️ Перезаписать** — заменить существующий рецепт
- **📝 Сохранить как новый** — создать копию с суффиксом

### Группы

Группы позволяют совместно собирать рецепты с другими пользователями.

#### Личная коллекция

Каждый пользователь автоматически получает личную группу «Личные рецепты». Она всегда доступна и не может быть удалена.

#### Создание группы

1. Нажмите **👥 Мои группы**
2. Нажмите **➕ Создать группу**
3. Введите название (до 50 символов)

Группа автоматически станет активной — новые рецепты будут сохраняться в неё.

#### Приглашение участников

1. Откройте группу (владелец)
2. Нажмите **🔗 Получить инвайт-код**
3. Передайте код участнику

#### Вступление в группу

1. Нажмите **👥 Мои группы**
2. Нажмите **🔗 Вступить по коду**
3. Введите полученный инвайт-код

#### Переключение между группами

Нажмите **👥 Мои группы** → выберите группу → **✅ Переключиться**. Все новые рецепты будут сохраняться в активную группу.

#### Выход из группы

Откройте группу (не личную) → **🚪 Покинуть группу**. Если активная группа была этой, бот автоматически переключится на личную.

> Владелец не может покинуть свою группу.

### Ограничения

- Длина видео — **до 3 минут**
- Минимальное количество распознанных слов — **10** (иначе бот решит, что речь не найдена)
- Таймаут обработки — **2 минуты**


## Архитектура

```
kuharych_bot/
├── bot.py                  # Точка входа, запуск бота
├── config.py               # Конфигурация из .env
├── handlers/
│   ├── link.py             # Обработка ссылок, пайплайн обработки видео
│   ├── buttons.py          # Callback-кнопки (удаление, перемещение, перезапись)
│   ├── menu.py             # Меню категорий, случайный рецепт
│   └── groups.py           # Управление группами (FSM)
├── services/
│   ├── downloader.py       # Скачивание видео (yt-dlp + kkclip fallback)
│   ├── transcriber.py      # Транскрибация аудио через Groq Whisper
│   ├── recipe_parser.py    # Генерация рецепта через GLM-5
│   ├── gramax.py           # CRUD рецептов через GitHub Contents API
│   └── group_manager.py    # Управление пользователями/группами (SQLite)
├── models/
│   ├── recipe.py           # Модель рецепта
│   └── group.py            # Модели пользователя и группы
├── prompts/
│   └── system.txt          # Системный промпт для GLM-5
└── data/
    └── bot.db              # SQLite база данных
```

### Пайплайн обработки видео

```
Ссылка → yt-dlp (скачивание) → FFmpeg (аудио) → Groq Whisper (транскрибация)
       → GLM-5 (генерация рецепта) → GitHub API (сохранение .md)
       → SQLite (индексирование)
```

## Тестирование

### Локальное тестирование (Unit тесты)

Запуск unit тестов локально (без API ключей):

```bash
# Все unit тесты
pytest tests/unit/

# Конкретный файл
pytest tests/unit/test_lobstr.py

# С выводом
pytest tests/unit/ -v
```

Unit тесты используют моки и не требуют API ключей.

### Тестирование на сервере (Integration тесты)

Запуск всех тестов (включая integration) на сервере:

```bash
# Через Telegram (только для админов)
/run_tests              # Все тесты
/run_tests unit         # Только unit тесты
/run_tests lobstr       # Lobstr API тесты
/run_tests pipeline     # End-to-end пайплайн
/run_tests transcriber  # Транскрибация

# Через командную строку
TESTING_ENV=server pytest tests/
TESTING_ENV=server pytest tests/integration/test_lobstr_live.py
```

Integration тесты:
- Требуют API ключей (`LOBSTR_API_KEY`, `GROQ_API_KEY`, `GLM_API_KEY`)
- Запускаются только при `TESTING_ENV=server`
- Пропускаются локально автоматически
- Отправляют результаты в Telegram как файл

### Структура тестов

```
tests/
├── unit/                          # Unit тесты (моки, быстрые)
│   ├── test_models.py            # Recipe, Group модели
│   ├── test_lobstr.py            # Валидация URL
│   ├── test_parser.py            # Парсинг рецептов
│   └── test_downloader.py        # Загрузка видео
├── integration/                   # Integration тесты (реальные API)
│   ├── test_lobstr_live.py       # Lobstr API
│   ├── test_pipeline.py          # End-to-end пайплайн
│   └── test_transcriber_live.py  # Groq транскрибация
└── fixtures/                      # Тестовые данные
    └── README.md                 # Инструкция для test_audio.mp3
```

### Окружение тестирования

Переменная `TESTING_ENV` управляет поведением тестов:

- `local` (по умолчанию): Unit тесты, integration тесты пропускаются
- `server`: Все тесты, integration тесты с реальными API

### Админские команды

Команда `/run_tests` доступна только администраторам (настраивается через `ADMIN_IDS` в `.env`).

**Требуемые переменные окружения:**
- `ADMIN_IDS`: Список ID администраторов через запятую
- `TESTING_ENV`: `local` или `server` (по умолчанию `local`)
