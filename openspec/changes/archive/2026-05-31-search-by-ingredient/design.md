# Design: Поиск рецептов по ингредиенту

## Архитектура

### Уровень данных (SQLite)

Новая таблица `recipe_ingredients` — денормализованный индекс ингредиентов для быстрого поиска:

```
recipe_ingredients
┌────────────┬──────────┬──────────────────────┐
│ category   │ slug     │ ingredient           │
├────────────┼──────────┼──────────────────────┤
│ осн. блюдо │ sup-kur  │ куриная грудка 300г  │
│ осн. блюдо │ sup-kur  │ лук 1 шт             │
│ осн. блюдо │ sup-kur  │ морковь 1 шт         │
│ завтрак    │ omlet    │ яйца 3 шт            │
└────────────┴──────────┴──────────────────────┘
PRIMARY KEY (category, slug, ingredient)
```

**Почему отдельная таблица, а не колонка JSON:**
- `LIKE` по текстовой колонке работает надёжнее чем JSON-функции SQLite
- Нет нужды в FTS5 для текущего масштаба
- Простота запросов

### Поисковый запрос

```sql
SELECT DISTINCT ri.slug
FROM recipe_ingredients ri
INNER JOIN group_recipes gr
  ON gr.category = ri.category AND gr.slug = ri.slug
WHERE ri.ingredient LIKE '%{query}%'
  AND gr.group_id = ?
  AND ri.category = ?
```

JOIN с `group_recipes` гарантирует что найденный рецепт принадлежит группе пользователя.

### FSM (aiogram State)

```
SearchState
  └── waiting_for_ingredient  — ожидает ввод ингредиента от пользователя
```

Хранение промежуточных данных (ингредиент, group_id) — через in-memory cache (`services/cache.py`).

### Callback-протокол

```
srch:{cc}:{ck}   — выполнить поиск (выбор категории)
srnd:{cc}:{ck}   — другой результат поиска
```

Где `cc` — короткий код категории, `ck` — ключ кэша с данными поиска.

## Решения

### Почему поиск в SQLite, а не в GitHub

- GitHub API: N запросов для скачивания каждого файла → медленно, rate limit
- SQLite: один запрос, мгновенный результат

### Почему не полнотекстовый поиск (FTS5)

- Текущий масштаб рецептов (десятки-сотни) не требует FTS
- `LIKE '%query%'` достаточно для поиска подстроки
- Можно мигрировать на FTS5 позже если понадобится

### Почему индексируем при сохранении, а не лениво

- Простой код — сохранение и индексация в одном месте
- Нет фоновой синхронизации
- Начинаем с пустого индекса — только новые рецепты индексируются
