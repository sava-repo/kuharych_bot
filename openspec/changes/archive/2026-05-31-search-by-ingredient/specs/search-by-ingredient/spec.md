# Spec: search-by-ingredient

## Возможность
Поиск рецептов по ингредиенту с фильтром по категории.

## Поведение

### Запуск поиска
- Пользователь нажимает кнопку «🔍 Поиск» в ReplyKeyboard меню
- Бот переходит в FSM-состояние `SearchState.waiting_for_ingredient`
- Бот отправляет сообщение: "🔎 Введите ингредиент для поиска:"

### Ввод ингредиента
- Пользователь отправляет текст (например, «курица»)
- FSM сбрасывается
- Бот кэширует `{ingredient, group_id}` и показывает inline-клавиатуру с тремя категориями:
  - 🌅 Завтрак
  - 🍽 Основное блюдо
  - 🍰 Десерт

### Выбор категории
- Пользователь нажимает inline-кнопку категории
- Бот выполняет SQL-поиск: `recipe_ingredients JOIN group_recipes WHERE ingredient LIKE '%{query}%' AND category = ? AND group_id = ?`
- Если найдены рецепты — бот выбирает случайный, загружает контент из GitHub, форматирует и отправляет
- Если не найдены — бот отправляет: "📭 Рецептов с ингредиентом «{query}» в категории «{category}» не найдено"

### Кнопка «Другой рецепт»
- Под найденным рецептом — inline-кнопка «🎲 Другой рецепт» (callback `srnd:{cc}:{ck}`)
- Повторяет поиск с тем же ингредиентом, исключая текущий рецепт (или просто случайный из найденных)

### Кнопки под рецептом
- 🗑 Удалить — стандартная
- 🎲 Другой рецепт — поиск с тем же ингредиентом
- 📂 Другая категория — стандартная смена категории

## Условия

- Поиск работает только по рецептам, добавленным ПОСЛЕ деплоя (пустой индекс)
- Поиск ведётся по подстроке (`LIKE '%query%'`)
- Результаты ограничены рецептами активной группы пользователя
- Данные ингредиентов синхронизируются при сохранении/обновлении/удалении рецептов
