# Spec Delta: search-by-ingredient

> Переопределение разделов существующей спецификации `search-by-ingredient`.

## Поведение

### Выбор категории
- Пользователь нажимает inline-кнопку категории
- Бот лемматизирует запрос пользователя через `pymorphy2` (см. `services/lemmatizer.py`) — получает список нормальных форм (лемм)
- Если запрос состоит только из чисел / стоп-слов и список лемм пуст — бот приравнивает это к «ничего не найдено»
- Бот выполняет SQL-поиск: `recipe_ingredients JOIN group_recipes WHERE ingredient_lemmas LIKE '%{lemma}%' AND category = ? AND group_id = ?`. При нескольких леммах в запросе условия `LIKE` объединяются через `AND` (точный поиск по фразе)
- Если найдены рецепты — бот выбирает случайный, загружает контент из GitHub, форматирует и отправляет
- Если не найдены — бот отправляет: "📭 Рецептов с ингредиентом «{query}» в категории «{category}» не найдено"

## Условия

- Поиск работает только по рецептам, добавленным ПОСЛЕ деплоя, плюс одноразовый backfill существующих строк при первом запуске (миграция колонки `ingredient_lemmas`)
- Поиск морфологически устойчив к русскому языку: запрос «яйца» находит «10 яиц», «яйцо»; запрос «индейка» находит «филе индейки»; запрос «курица» находит «куриная грудка»
- Нормализация выполняется на стороне приложения через `pymorphy2.MorphAnalyzer` (singleton на процесс)
- При индексации сохраняются две колонки: `ingredient` (исходная строка) и `ingredient_lemmas` (пробел-разделённые леммы)
- Поиск ведётся по нормализованной строке (`LIKE '%lemma%'` по колонке `ingredient_lemmas`)
- Цифры, местоимения, предлоги, союзы и частицы исключаются из набора лемм
- Результаты ограничены рецептами активной группы пользователя
- Данные ингредиентов синхронизируются при сохранении/обновлении/удалении рецептов

## Миграция данных

- При первом запуске после деплоя в `recipe_ingredients` добавляется колонка `ingredient_lemmas TEXT NOT NULL DEFAULT ''`
- Все существующие строки с пустой колонкой заполняются лемматизированными значениями (одноразовый backfill)
- Миграция идемпотентна и безопасна для повторных запусков
