# Proposal: Морфологический поиск по ингредиентам

## Why

Текущая реализация поиска в `services/group_manager.py:search_recipes_by_ingredient()` использует `LIKE '%{query}%'` по «сырой» строке ингредиента (например, «10 яиц», «филе индейки»). Такой поиск не учитывает морфологию русского языка:

- Запрос «яйца» не находит рецепт с ингредиентом «10 яиц» (разные окончания).
- Запрос «индейка» не находит рецепт с ингредиентом «филе индейки» (родительный падеж).
- Запрос «курица» не находит рецепт с ингредиентом «куриная грудка» (прилагательное от существительного).

В результате пользователь не получает релевантные рецепты, хотя они есть в коллекции.

## What Changes

- Внедряется морфологическая нормализация на базе `pymorphy2` (русский язык).
- При индексации рецепта каждая строка ингредиента дополнительно сохраняется в виде списка лемм (нормальных форм): «10 яиц» → «десять яйцо», «филе индейки» → «филе индейка».
- При поиске запрос пользователя также лемматизируется: «яйца» → «яйцо», «индейка» → «индейка».
- SQL-поиск переключается с `ingredient LIKE '%{query}%'` на `ingredient_lemmas LIKE '%{lemma}%'`, что обеспечивает устойчивость к падежам, числам и формам.
- Добавляется одноразовая миграция существующих строк `recipe_ingredients` (backfill колонки `ingredient_lemmas`) при старте бота.

## Capabilities

### Modified Capabilities
- `search-by-ingredient`: добавлено требование морфологической устойчивости поиска (лемматизация запроса и индекса через pymorphy2).

## Impact

- **Новый файл**: `services/lemmatizer.py` — обёртка над `pymorphy2.MorphAnalyzer` (singleton) + функции `normalize_word()`, `lemmatize_text()`.
- **Изменяемый файл**: `services/database.py` — миграция схемы (ALTER TABLE `recipe_ingredients` ADD COLUMN `ingredient_lemmas`) + backfill.
- **Изменяемый файл**: `services/group_manager.py` — `index_recipe_ingredients()` сохраняет леммы, `search_recipes_by_ingredient()` лемматизирует запрос и ищет по `ingredient_lemmas`.
- **Изменяемый файл**: `requirements.txt` — добавляется зависимость `pymorphy2>=0.9.1`.
- **Тесты**: добавляются unit-тесты на `services/lemmatizer.py` и на кейсы «10 яиц» / «филе индейки».
- **Хранилище**: без деградации — обратная совместимость сохранена (колонка `ingredient` остаётся, леммы — дополнительная).

## Non-goals

- Полнотекстовый поиск (FTS5) по названиям рецептов и шагам.
- Поиск по синонимам («мясо» ↔ «говядина» / «свинина»).
- Вынос поиска во внешний сервис (MeiliSearch / Elasticsearch).
- Поддержка языков кроме русского.
