# Spec Delta: category-management

> Новая capability: управление категориями уровня группы и отвязка категории от идентичности рецепта.

## ADDED Requirements

### Requirement: Категории принадлежат группе
Система SHALL хранить категории как сущность уровня группы. Каждая группа SHALL иметь независимый набор категорий. Личная группа пользователя (`pers_{id}`) выступает его персональным набором категорий.

#### Scenario: Новый пользователь получает стартовый набор
- **WHEN** пользователь впервые обращается к боту и для него создаётся личная группа `pers_{id}`
- **THEN** в этой группе создаются категории «завтрак», «основное блюдо», «десерт»
- **AND** категория «основное блюдо» помечена как категория по умолчанию (`is_default`)

#### Scenario: Одноимённые категории в разных группах независимы
- **GIVEN** группа A и группа B, обе с категорией «супы»
- **WHEN** в группе A категорию «супы» переименовывают в «первые блюда»
- **THEN** в группе B категория остаётся с именем «супы»

### Requirement: Категория не является частью идентичности рецепта
Рецепт SHALL идентифицироваться суррогатным `recipe_id`, независимым от категории. Один и тот же рецепт (один `reel_id`) МОЖЕТ находиться в разных категориях у разных групп.

#### Scenario: Один рилс в разных категориях у разных пользователей
- **GIVEN** пользователь A (личная группа) и пользователь B (личная группа)
- **WHEN** оба добавляют один и тот же рилс
- **THEN** контент рецепта хранится однократно (один `recipe_id`)
- **AND** в группе A рецепт относится к категории, выбранной для группы A
- **AND** в группе B рецепт относится к категории, выбранной для группы B

### Requirement: Создание категории
Пользователь SHALL иметь возможность добавить категорию в активную группу.

#### Scenario: Успешное добавление категории
- **GIVEN** активная группа пользователя
- **WHEN** пользователь создаёт категорию с уникальным в группе именем длиной 1–50 символов
- **THEN** категория появляется в меню пользователя и доступна для назначения рецептам

#### Scenario: Дубликат имени в группе
- **WHEN** пользователь создаёт категорию с именем, уже существующим в этой группе
- **THEN** операция отклоняется с понятным сообщением об ошибке

#### Scenario: Превышение лимита категорий
- **GIVEN** в группе уже `MAX_CATEGORIES_PER_GROUP` категорий
- **WHEN** пользователь пытается создать ещё одну
- **THEN** операция отклоняется с сообщением о лимите

### Requirement: Переименование категории
Пользователь SHALL иметь возможность переименовать категорию в активной группе. Рецепты, находящиеся в этой категории, SHALL сохранить привязку к ней.

#### Scenario: Переименование сохраняет рецепты
- **GIVEN** категория «супы» с 3 рецептами
- **WHEN** пользователь переименовывает её в «первые блюда»
- **THEN** название категории меняется
- **AND** те же 3 рецепта остаются привязаны к этой категории

### Requirement: Удаление категории
Пользователь SHALL иметь возможность удалить категорию. Категория с признаком `is_default` НЕ ДОЛЖНА (MUST NOT) подлежать удалению. При удалении не-default категории все рецепты из неё SHALL быть перенесены в default-категорию той же группы.

#### Scenario: Защита категории по умолчанию
- **WHEN** пользователь пытается удалить категорию с `is_default=1`
- **THEN** операция отклоняется с понятным сообщением

#### Scenario: Удаление с переносом рецептов
- **GIVEN** в группе есть default-категория и категория «удаляемая» с 3 рецептами
- **WHEN** пользователь удаляет «удаляемую»
- **THEN** 3 рецепта оказываются в default-категории
- **AND** в других группах ничего не меняется

### Requirement: Изоляция перемещения рецепта
Перемещение рецепта между категориями SHALL затрагивать только текущую группу и НЕ ДОЛЖНО (MUST NOT) менять категорию этого рецепта в других группах.

#### Scenario: Перенос не влияет на другие группы
- **GIVEN** рецепт R присутствует в группе A в категории «X» и в группе B в категории «Y»
- **WHEN** участник группы A переносит R в категорию «Z»
- **THEN** в группе A рецепт R относится к категории «Z»
- **AND** в группе B рецепт R остаётся в категории «Y»

### Requirement: Определение категории при добавлении рецепта
При добавлении нового рецепта система SHALL передавать LLM список категорий активной группы. LLM SHALL выбрать одну из них; если уверенного выбора нет — используется категория с `is_default=1`. Если контент рецепта уже распознан (рилс уже известен системе), рецепт SHALL добавляться в активную группу под default-категорию без повторного обращения к LLM.

#### Scenario: Новый рилс категоризируется LLM
- **WHEN** пользователь добавляет ранее неизвестный рилс
- **THEN** LLM получает список категорий активной группы и выбирает одну
- **AND** при неуверенном выборе рецепт попадает в default-категорию

#### Scenario: Известный рилс добавляется под default
- **GIVEN** рилс уже обработан и существует `recipe_id`
- **WHEN** пользователь из другой группы добавляет тот же рилс
- **THEN** LLM не вызывается повторно
- **AND** рецепт добавляется в активную группу под default-категорию
