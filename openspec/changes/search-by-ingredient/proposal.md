# Proposal: Поиск рецептов по ингредиенту

## Проблема

Сейчас пользователь может получить рецепт только выбрав категорию (Завтрак / Основное блюдо / Десерт) — бот выдаёт случайный рецепт. Нет возможности найти рецепт по конкретному ингредиенту (например, «курица»).

## Решение

Добавить поиск рецептов по ингредиенту с фильтром по категории:

1. **Новая кнопка «🔍 Поиск»** в ReplyKeyboard меню
2. **FSM-флоу**: пользователь вводит ингредиент → выбирает категорию → получает случайный рецепт из найденных
3. **Индекс ингредиентов в SQLite** — новая таблица `recipe_ingredients` для быстрого поиска без обращений к GitHub
4. **Автоматическая индексация** при сохранении/обновлении рецептов

## Пользовательский флоу

```
[🔍 Поиск] → Ввести ингредиент → Выбрать категорию → Случайный рецепт
```

## Масштаб изменений

- `services/group_manager.py` — +1 таблица, +3 функции
- `handlers/menu.py` — +FSM, +кнопка, +3 обработчика
- `handlers/link.py` — +1 вызов при сохранении
- `handlers/buttons.py` — +индексация при overwrite/save-new/move/delete

## Не-цели

- Полнотекстовый поиск по названиям рецептов
- Умный морфологический поиск (STEM), хватит `LIKE`
- Миграция/индексация существующих рецептов (начинаем с пустого индекса)
